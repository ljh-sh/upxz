# v0.1.0 (2026-06-26)

## Major

- **Initial release.** upxz is a tiny single-binary file packer: one file in,
  one file out, magic-checked, zstd-packed. Two subcommands: `pack` / `unpack`.

## Decisions (ljh-sh/mneme#41)

- **Compression backend:** zstd only. `xz2` / liblzma are intentionally not
  dependencies, keeping the build a single binary with a permissive license
  story (Apache-2.0 + BSD/MIT) and no LGPL entanglement.
- **Compression level:** 3-tier named preset — `--fast` (zstd 1), default
  (zstd 3), `--best` (zstd 19). `--fast` and `--best` are mutually exclusive.

## Container format

```
magic (8 bytes) | name-len (4 bytes BE) | original file name (UTF-8) | zstd frame
```

- Magic: `UPXZ\x01\x00\x00\x00`.
- Stored file name is a flat path component; re-validated on unpack.
- Double-packing an existing container is refused.

## Security

- Flat-name validation prevents path traversal via a crafted header.
- Bounded name length guards against corrupted length fields.
- See `SECURITY.md`.
